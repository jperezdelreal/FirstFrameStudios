## Error Handling Patterns

### Pattern 1: API Try-Catch
```javascript
try {
  const response = await fetch(url, {signal: controller.signal});
  setConnectionState(ConnectionState.OPERATIONAL);
  return response;
} catch (error) {
  logError(context, error);
  setConnectionState(ConnectionState.DEGRADED | OFFLINE);
  throw error;
}
```

### Pattern 2: Component Error States
```javascript
async function refresh() {
  try {
    const data = await apiGet('/api/endpoint');
    renderComponent(data);
  } catch (error) {
    renderError(error);  // Shows error icon + retry button
  }
}
```

### Pattern 3: Exponential Backoff (SSE)
```
Delays: 1s → 2s → 4s → 8s → 30s
Max Retries: 10
After max, shows error state with manual retry
```

```javascript
// Studio Pulse: Shows "activity unavailable"
// Auto-retries without blocking UI
// No manual retry button needed
```

# Ashfall — Sprint Roadmap

> Master roadmap for Ashfall development. Updated each sprint.

## Vision
1v1 fighting game in Godot 4. 2 characters, 1 stage, local play. "Guilty Gear quality" visual bar.

## Sprint Overview

| Sprint | Focus | Status | Tag |
|--------|-------|--------|-----|
| Sprint 0 | Foundation — architecture, game loop, state machine, basic combat | ✅ Shipped | ashfall-v0.1.0 |
| Sprint 1 | Art Phase — character sprites, stage art, animation integration, VFX | ✅ Shipped | ashfall-v0.2.0 |
| Sprint 2 | Visual Quality — code hardening, camera, sprites, stage, HUD, AI, combat feel | ✅ Shipped | ashfall-v0.3.0 |
| Sprint 3 | Audio — SFX, procedural music, announcer voice, character audio palettes | 📋 Planned | — |
| Sprint 4 | Polish & Balance — training mode, balance tuning, accessibility, polish | 📋 Planned | — |
| Sprint 5+ | Expansion — additional characters, stages, online multiplayer, story mode | 📋 Future | — |

## Sprint 3: Audio (Next)
**Focus:** Sound design and music. The game should SOUND like a fighting game.

Planned work:
- Hit SFX (light/medium/heavy impacts, blocks, whiffs)
- Character-specific audio palettes (Kael: fire crackle, Rhena: wind whoosh)
- Procedural music system (3 intensity levels matching round progression)
- Announcer voice ("ROUND 1", "FIGHT!", "K.O." callouts)
- Menu/UI sound effects
- Owner: Greedo (Sound Designer)

## Sprint 4: Polish & Balance
**Focus:** Game feel refinement and training features.

Planned work:
- Training mode with frame data overlay
- Balance tuning (damage, hitstun, combo proration)
- Character select vs_player mode
- Input display / combo trial mode
- Accessibility options
- Performance optimization

## Sprint 5+: Expansion
**Focus:** Content and multiplayer.

Planned work:
- Additional characters (2-4 more)
- Additional stages (2-3 more)
- Online multiplayer (rollback netcode)
- Story/arcade mode with endings
- Replay system

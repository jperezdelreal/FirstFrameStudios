# Spec Verification Checklist

Before starting implementation, verify:

- [ ] **Clear Acceptance Criteria** — Issue has specific, measurable acceptance criteria (not vague or open-ended)
- [ ] **Dependencies Identified** — Check for blocking issues or unmet requirements; note in comment if dependencies exist
- [ ] **Target Files Exist** — Verify that the files/systems referenced in the spec already exist or are correctly scoped
- [ ] **No Conflicting Work** — Search for in-progress issues on the same feature; confirm with team if ambiguous
- [ ] **Context Available** — GDD sections, architecture docs, or design docs are referenced (where applicable)

**Time: ~30 seconds to verify**

If any item is unclear, comment on the issue to request clarification before implementation.

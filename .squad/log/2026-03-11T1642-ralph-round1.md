## Team Decisions Made

1. **Parallel Code Review + QA:** Solo and Ackbar work concurrently to gather feedback faster than sequential gates. Reduces round-trip delays.

2. **Infrastructure-First ffs-squad-monitor:** Jango and Wedge implement monitoring core before Ralph expands orchestration. Enables observability from Round 2 onward.

3. **Heartbeat Protocol (Issue #1):** Simple JSON polling (vs. WebSocket) for MVP—easier to implement, debug, and extend.

4. **Dashboard Architecture (Issue #4):** Consume heartbeat via REST endpoint; stateless UI layer. Decouples UI from monitoring logic.

5. **Non-Blocking Tasks:** All four agents operate independently; no ordering dependencies. Maximizes throughput.

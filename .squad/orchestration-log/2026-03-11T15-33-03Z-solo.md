## Deliverables

✅ **7-Module Architecture**
- `core/`: SceneManager, EventBus, (future: input, asset loader)
- `scenes/`: Individual game screens (boot, menu, garden, etc.)
- `entities/`: Game objects (plants, player, tools)
- `systems/`: ECS-lite update loops (growth, weather, inventory)
- `ui/`: HUD, menus, dialogs
- `utils/`: Pure functions, math, RNG helpers
- `config/`: Constants, balance values

✅ **Key Patterns**
- Scene-Based Architecture: One active at a time; clean init/update/destroy
- ECS-Lite: Lightweight systems, no full framework overhead
- Typed Event Bus: Pub-sub with EventMap for compile-time safety
- PixiJS v8 API: `Application.init()`, `app.canvas`, `Text({text, style})`, `Assets.load()`

✅ **Dependency Direction**
- main.ts → core → scenes → entities/systems/ui
- config, utils ← imported by anything
- EventBus ← imported by scenes, systems, ui

✅ **Rationale Documented**
- Modular from day one prevents ComeRosquillas monolith anti-pattern
- Scene-based simpler than FSM for small-medium game
- ECS-lite avoids framework overhead while keeping separation of concerns

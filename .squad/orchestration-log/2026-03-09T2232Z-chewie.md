### Implementation Details

**Scene Structure:**
- Node2D root with script attached
- All nodes created programmatically in `_ready()`
- AnimatedSprite2D with Embergrounds background
- HUD overlay for current animation state display

**Animation Configuration (Data-Driven):**
```
ANIM_CONFIG = {
  "idle": {"frames": 8, "fps": 8, "loop": true},
  "walk": {"frames": 8, "fps": 10, "loop": true},
  "lp": {"frames": 12, "fps": 60, "loop": false}
}
```

**Control Scheme:**
- `1` — Play idle animation
- `2` — Play walk animation
- `3` — Play LP (light punch) animation
- `F` — Toggle fullscreen
- `ESC` — Exit

**Visual Configuration:**
- Center-bottom origin: `offset = Vector2(0, -256)`
- Scale: 0.4 (renders 512×512 sprites at ~205px on 1080p)
- Texture filter: `TEXTURE_FILTER_NEAREST` for pixel-perfect rendering
- Background: Embergrounds 1920×1080

### Design Principles

1. **Fully Programmatic** — No manual scene editor work; changes tracked via version control
2. **Data-Driven Config** — Adding animations requires only config entry and PNG files
3. **Self-Contained** — Runtime texture filtering doesn't affect production assets
4. **Fighting Game Convention** — Character position represents ground contact point
5. **Auto-Return** — Non-looping animations return to idle via `animation_finished` signal

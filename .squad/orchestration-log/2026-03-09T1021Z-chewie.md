## Task Summary

Fix P1 #95 — equal-HP timer draw infinite loop. When both fighters reach 0 HP simultaneously, round_manager loops forever instead of declaring round_draw and proceeding.

## Outcome

**Bug fixed. PR #97 merged to main.** RoundManager now correctly handles draw states with new signal architecture.

### Work Completed

1. **Draw State Logic Fix**
   - Identified root cause: `check_round_end()` only checked for KO (HP < 0), not equal HP depletion
   - Added check: if both fighters' HP ≤ 0, emit `round_draw` signal instead of entering KO state
   - Draw counts as completion (winner = nil, advances to next round or rematch)

2. **Signal Architecture**
   - **New signal:** `round_draw` — emitted when round ends in draw (equal-HP depletion)
   - **New signal:** `match_draw` — emitted when entire match ends in draw (all rounds drawn, not applicable for 1v1)
   - Existing `round_end` signal now guaranteed to emit after `round_draw`

3. **GDD Alignment**
   - Per GDD: "Equal HP = mutual KO = draw round"
   - RoundManager now correctly implements this per-round
   - Matches arcade fighting game conventions (simultaneous KOs are draws)

## Key Lesson Learned

**Game rules must be explicit in code.** The GDD said "equal HP = draw," but the round_manager logic only knew about KO. Encoding rules as signals makes them debuggable and testable independently.

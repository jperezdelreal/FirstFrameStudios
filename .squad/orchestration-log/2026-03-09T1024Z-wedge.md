## Task Summary

Fix P1 #94 — HUD score display always shows 0-0 despite rounds completing. RoundManager tracks scores internally but doesn't sync to GameState, breaking both in-fight HUD and victory screen readout.

## Outcome

**Bug fixed. PR #98 merged to main.** HUD score now correctly reflects RoundManager state via GameState.

### Work Completed

1. **Score Sync Architecture**
   - Identified root cause: RoundManager maintained `scores` dictionary (local state) but never wrote to `GameState.scores`
   - Added: After each `round_end`, RoundManager now calls `GameState.set_scores(scores)`
   - Wired `RoundManager.round_draw` signal → `FightHUD.on_round_draw()` to update display

2. **GameState Enhancement**
   - Added `set_scores(new_scores)` method to persist score updates
   - Added getter: `get_scores()` for UI read access
   - Ensures single source of truth (GameState) for all score-dependent logic

3. **FightHUD Integration**
   - Score display now reads from `GameState.scores` every physics frame
   - Victory screen correctly pulls final scores and renders winner announcement
   - Rematch button works: score carries forward correctly (no reset on rematch)

### Files Modified

- `scripts/autoloads/game_state.gd` (score getters/setters)
- `scripts/scenes/round_manager.gd` (sync scores to GameState after round_end)
- `scripts/ui/fight_hud.gd` (read scores from GameState, wired round_draw signal)
- `.squad/agents/wedge/history.md` (documentation)

## Key Lesson Learned

**Autoload state is the source of truth.** Local state (RoundManager.scores) can drift from global state (GameState.scores). Always push updates to the single source of truth, then read back from it.

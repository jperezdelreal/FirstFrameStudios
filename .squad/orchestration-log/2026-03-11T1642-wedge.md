## Task Specification

### Issue #4 — Dashboard UI (P1)

**Description:**  
Build a real-time monitoring dashboard displaying team agent status, active tasks, and system health. Consumes heartbeat data from Jango's Issue #1 reader.

**Scope:**
- [ ] Dashboard layout (agent status grid, real-time updates, refresh indicators)
- [ ] Agent status cards (name, status badge, last task, uptime)
- [ ] System health summary (online count, stalled count, overall status)
- [ ] Real-time updates via polling (sync with Jango's heartbeat endpoint)
- [ ] Responsive design (desktop, tablet, mobile viewport support)
- [ ] Light/dark mode toggle
- [ ] Unit test coverage (80%+ target)

**Architecture Notes:**
- Consume `GET /api/heartbeat` endpoint from Jango's heartbeat reader
- Poll interval: 5s (match heartbeat reader)
- Status states: `online`, `idle`, `stalled`, `offline`
- Color scheme: green (healthy), yellow (idle/degraded), red (stalled/offline)
- Layout should scale to 10-20 agents without performance degradation

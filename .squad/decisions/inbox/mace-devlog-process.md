# Dev Diary Post Process

**Decision**: Mace should create a Dev Diary discussion post after each milestone completion.

## Process

After each milestone (M1, M2, M3, M4, etc.) is completed and merged:

1. **Timing**: Post the devlog within 24 hours of milestone completion
2. **Category**: Post in the "General" discussion category
3. **Title Format**: `🔥 Dev Diary #X: [Milestone Title]` (e.g., "#2: Character Sprites & Polish")
4. **Content Structure**:
   - **The Pitch** (brief intro to Ashfall and Ember System)
   - **What We Shipped Today** (M{X} work, PRs merged, key features)
   - **By The Numbers** (PR count, GDScript files, issues closed, lines of code)
   - **What's Next** (preview of upcoming milestone)
   - **The Meta** (AI-powered development commentary)
   - **CTA**: "Follow this repo to watch the studio work..."

5. **Tone**: Passionate, transparent, slightly irreverent. Indie dev blog meets behind-the-scenes documentary. NOT corporate.

6. **Visibility**: These posts are public-facing marketing for First Frame Studios. Every post should remind readers that this is an AI-powered studio shipping real game code.

## Integration with Wiki

- Update `.squad/wiki/milestones.md` after each milestone
- Link the Dev Diary discussion from the milestone entry
- Track discussion URL for metrics/engagement
